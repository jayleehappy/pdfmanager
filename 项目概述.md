# 项目概述

## 项目背景
领导干部个人有关事项报告表 OCR 识别与查核反馈比对系统。

扫描件为PDF格式，与21页标准模板逐一比对，识别报告表中的填写内容。

## 核心问题
扫描件页数可能与模板不一致（空白页、缺失页、跳页），需要先做**页匹配**，确定每个扫描页对应哪个模板页。

## 技术栈
- **OCR引擎**: PaddleOCR-json (C++ 子进程, MKL-DNN 加速) + RapidOCR (PreOCR 模板匹配)
- **Web框架**: FastAPI + Uvicorn
- **PDF处理**: PyMuPDF (fitz)
- **特征匹配**: ORB特征点匹配
- **预OCR**: RapidOCR 识别特征区域文本，实现快速页匹配
- **区域识别**: PaddleOCR-json 识别模板标注的手写内容区域

## 模板结构
| 模板页 | 内容 | 预OCR特征 |
|--------|------|-----------|
| 1 | 封面标题 | "领导干部个人有关事项报告表" |
| 2 | 填表须知 | "填表须知" |
| 3 | 目录 | "目录" |
| 4-21 | 报告表各页 | 页码 "1"~"18" |

## 公共特征区域（`templates/pre_ocr_region.json`）
三个独立小区域（已废弃合并拼接，改为分步独立识别）：
- **区域1 (idx=1)**: 标题区 `x=263~997 y=343~460` (734×117px) → Step2
- **区域2 (idx=2)**: 页脚区 `x=12~1216 y=1628~1732` (1204×104px) → **Step1核心**
- **区域3 (idx=3)**: 说明区 `x=376~880 y=125~249` (504×124px) → Step3

## 匹配策略（PreOCR 三步级联）

**Step 1**：仅识别页脚区（y≈1628-1732，约104px高）→ 页码数字 → 模板4-21
**Step 2**：识别标题区（y≈343-460）→ "领导干部个人..." → 模板1 / "目录" → 模板3
**Step 3**：识别说明区（y≈125-249）→ "填表须知" → 模板2
**兜底**：ORB 特征匹配（极少触发）

详见 `api/ocr_preocr.py` 的 `_extract_page_num()` 和三步级联逻辑。

### 性能指标
- PreOCR（19页）：~55s（仅页脚区）
- 正式OCR（55个区域）：~2.7s（PaddleOCR-json via 127.0.0.1:1224）
- 总耗时：~60s/扫描件

### OCR识别噪声处理
OCR 输出页码可能含汉字（如 `"7 一"`、`"— 10 — 一"`），用正则去除：
```python
re.sub(r'[\u4e00-\u9fff\u3000\xa0\u2014-]+', '', text)
```
